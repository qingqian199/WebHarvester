# WebHarvester 操作界面重新设计

## 设计目标

借鉴 lazygit / k9s / gh 等工具的成熟模式：
- **状态栏**始终可见，不占菜单空间
- **渐进式呈现**：高频操作放在最前，高级功能折叠
- **上下文感知**：根据系统状态动态显示选项
- **最少输入**：默认值自动填充，减少用户键盘操作

---

## 当前问题

```
问题1: 14 个菜单项平铺，高频低频混在一起
问题2: 每次操作需要多次 inquirer 问答（输入URL→选会话→确认...）
问题3: 状态信息在菜单标题中，不直观
问题4: 操作反馈不足（采集完只打印几行，没有统一结果页）
```

### 采集流程（简化后）

#### 当前: 5 次问答
```
选择采集模式 → 输入URL → 配置选项 → 选择会话 → 确认
```

#### 新设计: 1-2 次问答
```
输入 URL 后自动:
  1. 匹配特化爬虫 → 有则直接用 API + 签名
  2. 无匹配 → 通用浏览器引擎
  3. 检测到登录会话 → 自动附加 Cookie
  4. 检测到 CDP → 可用 CDP 增强

用户只需: 输入 URL → [Enter] → 等结果
```

## 核心改进对比

| 维度 | 当前 | 新设计 |
|------|------|--------|
| 采集操作步骤 | 5 次问答 | 1-2 次 |
| 菜单层级 | 1 级 14 项 | 2 级，按频率分组 |
| 状态显示 | 菜单标题中 | 独立状态栏 + 颜色标记 |
| 结果反馈 | 纯文本打印 | 结构化结果页 + 操作按钮 |
| 会话管理 | 文本列表 | 状态标记（有效/过期）+ 批量操作 |
| 键盘操作 | 仅数字选择 | 数字 + 快捷键（s/d/q/r） |
| 配置查看 | 单独菜单项 | 快捷键直达 |

## 实现策略

1. **保留 inquirer**（已有依赖，不需要换库）
2. **新增 `src/cli/ui/` 目录**：
   - `status-bar.ts` — 顶部状态栏
   - `result-page.ts` — 采集结果展示
   - `session-manager.ts` — 会话管理界面
3. **重构 `main-menu.ts`** — 精简主菜单，上下文感知
4. **渐进实施** — 先改主菜单结构，再增强结果页
